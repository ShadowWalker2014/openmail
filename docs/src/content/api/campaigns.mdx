# Campaigns API

Campaigns are automated email sequences triggered by customer events or segment membership changes. When a trigger fires, enrolled contacts move through the campaign's steps.

**Base URL:** `/api/v1/campaigns`

## List Campaigns

<EndpointBadge method="GET" path="/api/v1/campaigns" />

Returns all campaigns in the workspace.

## Get Campaign

<EndpointBadge method="GET" path="/api/v1/campaigns/:id" />

Returns a campaign with its steps included.

```json
{
  "id": "cmp_abc123def456",
  "name": "Welcome Series",
  "status": "active",
  "triggerType": "event",
  "triggerConfig": { "eventName": "user_signed_up" },
  "steps": [
    {
      "id": "stp_xxx",
      "campaignId": "cmp_abc123def456",
      "stepType": "email",
      "position": 0,
      "config": { "templateId": "tpl_welcome", "subject": "Welcome!" }
    },
    {
      "id": "stp_yyy",
      "campaignId": "cmp_abc123def456",
      "stepType": "wait",
      "position": 1,
      "config": { "delaySeconds": 259200 }
    }
  ]
}
```

## Create Campaign

<EndpointBadge method="POST" path="/api/v1/campaigns" />

Creates a new campaign in `draft` status. A campaign cannot be activated until it has at least one step.

### Request Body

| Field | Type | Required | Description |
|-------|------|:--------:|-------------|
| `name` | string | ✓ | Campaign name |
| `triggerType` | enum | ✓ | What triggers this campaign (see below) |
| `triggerConfig` | object | * | Trigger configuration — required fields depend on trigger type |
| `description` | string | — | Optional description |

### Trigger Types

| `triggerType` | Description | Required `triggerConfig` |
|---------------|-------------|--------------------------|
| `event` | Fires when a contact triggers a named event | `{ "eventName": "user_signed_up" }` |
| `segment_enter` | Fires when a contact joins a segment | `{ "segmentId": "seg_xxx" }` |
| `segment_exit` | Fires when a contact leaves a segment | `{ "segmentId": "seg_xxx" }` |
| `manual` | Triggered via API only | `{}` |

```bash
curl -X POST https://your-api/api/v1/campaigns \
  -H "Authorization: Bearer om_..." \
  -H "Content-Type: application/json" \
  -d '{
    "name": "Welcome Series",
    "triggerType": "event",
    "triggerConfig": { "eventName": "user_signed_up" },
    "description": "Onboard new users"
  }'
```

## Update Campaign

<EndpointBadge method="PATCH" path="/api/v1/campaigns/:id" />

Updates a campaign's name, description, or status.

### Status Transitions

Campaigns follow a strict state machine:

```
draft → active ⇄ paused
     ↘          ↘
      archived   archived (terminal — cannot undo)
```

- `draft → active`: Activates the campaign (**requires at least one step**)
- `active → paused`: Stops enrolling new contacts
- `paused → active`: Resumes the campaign
- Any → `archived`: Terminal state — cannot be reactivated

```bash
# Activate a campaign
curl -X PATCH https://your-api/api/v1/campaigns/cmp_abc123def456 \
  -H "Authorization: Bearer om_..." \
  -H "Content-Type: application/json" \
  -d '{ "status": "active" }'
```

## Delete Campaign

<EndpointBadge method="DELETE" path="/api/v1/campaigns/:id" />

Permanently deletes a campaign and all its steps. Returns `204 No Content`.

## Campaign Steps

Campaign steps define what happens to an enrolled contact. Steps execute in order by `position`.

### Step Types

| `stepType` | Description | Config fields |
|------------|-------------|---------------|
| `email` | Send an email | `{ "templateId": "tpl_xxx", "subject": "...", "fromEmail": "...", "fromName": "..." }` |
| `wait` | Delay before next step | `{ "delaySeconds": 259200 }` (e.g. 259200 = 3 days) |

### Add Step

<EndpointBadge method="POST" path="/api/v1/campaigns/:id/steps" />

Adds a step to a campaign. **Campaign must be in `draft` status.**

| Field | Type | Required | Description |
|-------|------|:--------:|-------------|
| `stepType` | `"email"` or `"wait"` | ✓ | Type of step |
| `config` | object | — | Step configuration (see step types above) |
| `position` | integer | — | Zero-based position (appended to end if omitted) |

```bash
# Add an email step
curl -X POST https://your-api/api/v1/campaigns/cmp_abc123def456/steps \
  -H "Authorization: Bearer om_..." \
  -H "Content-Type: application/json" \
  -d '{
    "stepType": "email",
    "config": {
      "templateId": "tpl_welcome",
      "subject": "Welcome to Acme!"
    }
  }'

# Add a 3-day wait step
curl -X POST https://your-api/api/v1/campaigns/cmp_abc123def456/steps \
  -H "Authorization: Bearer om_..." \
  -H "Content-Type: application/json" \
  -d '{
    "stepType": "wait",
    "config": { "delaySeconds": 259200 }
  }'
```

### Update Step

<EndpointBadge method="PATCH" path="/api/v1/campaigns/:id/steps/:stepId" />

Updates a step's config or position.

| Field | Type | Description |
|-------|------|-------------|
| `config` | object | New step configuration |
| `position` | integer | New zero-based position |

### Delete Step

<EndpointBadge method="DELETE" path="/api/v1/campaigns/:id/steps/:stepId" />

Permanently deletes a step from the campaign. Returns `204 No Content`.

## Campaign Object

| Field | Type | Description |
|-------|------|-------------|
| `id` | string | Unique ID (`cmp_` prefix) |
| `name` | string | Campaign name |
| `description` | string or null | Optional description |
| `status` | string | `draft`, `active`, `paused`, or `archived` |
| `triggerType` | string | Event or segment-based trigger |
| `triggerConfig` | object | Trigger-specific configuration |
| `steps` | array | Included when fetching by ID |
| `createdAt` | timestamp | Creation time |
| `updatedAt` | timestamp | Last update time |
