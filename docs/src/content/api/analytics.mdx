# Analytics API

The Analytics API provides aggregate performance metrics for your workspace and individual broadcasts.

**Base URL:** `/api/v1/analytics`

## Workspace Overview

<EndpointBadge method="GET" path="/api/v1/analytics/overview" />

Returns workspace-wide metrics for the last 30 days.

```bash
curl -H "Authorization: Bearer om_..." \
  https://your-api/api/v1/analytics/overview
```

### Response

```json
{
  "totalContacts": 12453,
  "newContactsLast30Days": 847,
  "totalSends": 58920,
  "openRate": 0.243,
  "clickRate": 0.051,
  "unsubscribeRate": 0.008,
  "periodStart": "2025-01-15T00:00:00.000Z",
  "periodEnd": "2025-02-14T23:59:59.000Z"
}
```

### Response Fields

| Field | Type | Description |
|-------|------|-------------|
| `totalContacts` | integer | Total contacts in workspace |
| `newContactsLast30Days` | integer | Contacts created in the last 30 days |
| `totalSends` | integer | Emails sent in the last 30 days |
| `openRate` | float | Open rate as a decimal (0–1) |
| `clickRate` | float | Click-through rate as a decimal (0–1) |
| `unsubscribeRate` | float | Unsubscribe rate as a decimal (0–1) |
| `periodStart` | timestamp | Start of the 30-day window |
| `periodEnd` | timestamp | End of the 30-day window |

## Broadcast Analytics

<EndpointBadge method="GET" path="/api/v1/analytics/broadcasts/:id" />

Returns detailed performance metrics for a specific broadcast.

```bash
curl -H "Authorization: Bearer om_..." \
  https://your-api/api/v1/analytics/broadcasts/brd_abc123def456
```

### Response

```json
{
  "broadcastId": "brd_abc123def456",
  "name": "March Newsletter",
  "subject": "What's new in March 🌱",
  "sentAt": "2025-03-01T09:00:00.000Z",
  "recipientCount": 4821,
  "sentCount": 4812,
  "openCount": 1169,
  "clickCount": 245,
  "unsubscribeCount": 12,
  "bounceCount": 9,
  "openRate": 0.243,
  "clickRate": 0.051,
  "unsubscribeRate": 0.0025
}
```

### Response Fields

| Field | Type | Description |
|-------|------|-------------|
| `broadcastId` | string | Broadcast ID |
| `recipientCount` | integer | Total intended recipients |
| `sentCount` | integer | Emails actually sent |
| `openCount` | integer | Unique opens tracked |
| `clickCount` | integer | Unique clicks tracked |
| `unsubscribeCount` | integer | Unsubscribes from this broadcast |
| `bounceCount` | integer | Hard + soft bounces |
| `openRate` | float | Opens / sentCount |
| `clickRate` | float | Clicks / sentCount |

## Tracking Notes

Open and click tracking require the `tracker/` service to be deployed and `TRACKER_URL` set in the API environment. Without the tracker:
- `openCount` and `clickCount` will be 0
- `openRate` and `clickRate` will be 0
- All other fields are accurate

Open tracking works by embedding a 1×1 transparent pixel in each email. Some email clients block pixels, so actual open rates may be higher than reported.
