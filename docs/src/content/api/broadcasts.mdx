# Broadcasts API

Broadcasts are one-off email blasts sent to one or more contact segments. A broadcast starts as a draft, then can be sent immediately or scheduled for a future time.

**Base URL:** `/api/v1/broadcasts`

## List Broadcasts

<EndpointBadge method="GET" path="/api/v1/broadcasts" />

Returns all broadcasts in the workspace.

```bash
curl -H "Authorization: Bearer om_..." \
  https://your-api/api/v1/broadcasts
```

### Response

```json
[
  {
    "id": "brd_abc123def456",
    "workspaceId": "ws_xyz789",
    "name": "March Newsletter",
    "subject": "What's new in March 🌱",
    "status": "sent",
    "segmentIds": ["seg_abc123"],
    "recipientCount": 4821,
    "sentCount": 4812,
    "scheduledAt": null,
    "sentAt": "2025-03-01T09:00:00.000Z",
    "createdAt": "2025-02-28T14:00:00.000Z"
  }
]
```

## Get Broadcast

<EndpointBadge method="GET" path="/api/v1/broadcasts/:id" />

Returns a single broadcast with full details.

## Create Broadcast

<EndpointBadge method="POST" path="/api/v1/broadcasts" />

Creates a new broadcast in `draft` status. The broadcast is not sent until you call the send endpoint.

### Request Body

| Field | Type | Required | Description |
|-------|------|:--------:|-------------|
| `name` | string | ✓ | Internal name (not shown to recipients) |
| `subject` | string | ✓ | Email subject line |
| `segmentIds` | string[] | ✓ | One or more segment IDs to send to |
| `htmlContent` | string | * | Full HTML email body |
| `templateId` | string | * | Saved template ID — use instead of `htmlContent` |
| `fromEmail` | string | — | Sender address (falls back to workspace default) |
| `fromName` | string | — | Sender display name |
| `scheduledAt` | ISO datetime | — | Schedule for future delivery |

\* Either `htmlContent` or `templateId` must be provided, not both.

```bash
curl -X POST https://your-api/api/v1/broadcasts \
  -H "Authorization: Bearer om_..." \
  -H "Content-Type: application/json" \
  -d '{
    "name": "Welcome Series",
    "subject": "Welcome to Acme! 👋",
    "segmentIds": ["seg_abc123"],
    "htmlContent": "<h1>Welcome!</h1><p>Thanks for joining.</p>",
    "fromEmail": "hello@acme.com",
    "fromName": "Acme Team"
  }'
```

## Update Broadcast

<EndpointBadge method="PATCH" path="/api/v1/broadcasts/:id" />

Updates a broadcast that is still in `draft` status. Cannot update a broadcast that is `sending` or `sent`.

All fields are optional — only provided fields are updated.

## Send Broadcast

<EndpointBadge method="POST" path="/api/v1/broadcasts/:id/send" />

Immediately queues the broadcast for sending. This action is irreversible.

```bash
curl -X POST https://your-api/api/v1/broadcasts/brd_abc123def456/send \
  -H "Authorization: Bearer om_..."
```

The broadcast status changes to `queued`, then `sending`, then `sent`. Send progress is tracked via `sentCount` / `recipientCount`.

## Schedule Broadcast

<EndpointBadge method="POST" path="/api/v1/broadcasts/:id/schedule" />

Schedules a broadcast for future delivery.

```bash
curl -X POST https://your-api/api/v1/broadcasts/brd_abc123def456/schedule \
  -H "Authorization: Bearer om_..." \
  -H "Content-Type: application/json" \
  -d '{ "scheduledAt": "2025-03-15T09:00:00Z" }'
```

## Delete Broadcast

<EndpointBadge method="DELETE" path="/api/v1/broadcasts/:id" />

Permanently deletes a draft broadcast. Cannot delete a broadcast that has been sent or is currently sending.

## Broadcast Statuses

| Status | Description |
|--------|-------------|
| `draft` | Created but not yet sent or scheduled |
| `scheduled` | Queued for future delivery |
| `queued` | Submitted to the send queue |
| `sending` | Currently being processed by workers |
| `sent` | All emails delivered to the queue |
| `failed` | Send failed — check worker logs |

## Broadcast Object

| Field | Type | Description |
|-------|------|-------------|
| `id` | string | Unique ID (`brd_` prefix) |
| `name` | string | Internal broadcast name |
| `subject` | string | Email subject line |
| `status` | string | Current status (see above) |
| `segmentIds` | string[] | Target segment IDs |
| `htmlContent` | string or null | Inline HTML content |
| `templateId` | string or null | Referenced template ID |
| `fromEmail` | string or null | Sender address |
| `fromName` | string or null | Sender name |
| `recipientCount` | integer | Total unique recipients |
| `sentCount` | integer | Emails sent so far |
| `scheduledAt` | timestamp or null | Scheduled send time |
| `sentAt` | timestamp or null | When sending completed |
