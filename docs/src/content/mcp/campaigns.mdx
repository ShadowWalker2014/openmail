# Campaign Tools

8 tools across two namespaces:

- **`campaigns.*`** — CRUD: list, create, update, pause (deprecated)
- **`lifecycle.*`** — verb-style lifecycle control with full audit trail (Stage 2)

> ⚠️ **Deprecation notice** — `update_campaign({ status: ... })` and the legacy `pause_campaign` tool are kept for backward compatibility but should NOT be used by new agents. Use the `lifecycle.*` tools below — they emit a full audit trail and expose explicit drain / force / archive verbs with destructive-action confirmations.

## list_campaigns

List all automation campaigns in the workspace.

> "Show me all my campaigns and their status"

## create_campaign

Create a new automation campaign.

| Parameter | Type | Required | Description |
|-----------|------|:--------:|-------------|
| `name` | string | ✓ | Campaign name |
| `triggerType` | string | ✓ | `event`, `segment_enter`, `segment_exit`, or `manual` |
| `triggerConfig` | object | * | Trigger configuration (see below) |
| `description` | string | — | Optional description |

`triggerConfig` by type:
- `event`: `{ "eventName": "user_signed_up" }`
- `segment_enter`: `{ "segmentId": "seg_xxx" }`
- `segment_exit`: `{ "segmentId": "seg_xxx" }`
- `manual`: `{}`

> "Create a welcome campaign that fires when someone signs up. Make it active."

```
Tool: create_campaign
{
  "name": "Welcome Series",
  "triggerType": "event",
  "triggerConfig": { "eventName": "user_signed_up" }
}
→ returns campaign with id: cmp_xxx (status: draft)

Tool: update_campaign
{ "campaignId": "cmp_xxx", "status": "active" }
```

| Parameter | Type | Required | Description |
|-----------|------|:--------:|-------------|
| `campaignId` | string | ✓ | Campaign ID (`cmp_xxx`) |
| `name` | string | — | New campaign name |
| `description` | string | — | New description |
| `status` | string | — | **[Deprecated]** `active`, `paused`, or `archived` |

## Lifecycle Tools <StabilityBadge level="stable" />

The `lifecycle.*` family wraps the verb endpoints with full audit trail and destructive-action confirmations.

## lifecycle.pause_campaign (Stage 2) <StabilityBadge level="stable" />

Pause an active campaign — stops new enrollments and freezes pending wait jobs. Reversible: call `resume_campaign` to continue.

| Parameter | Type | Required | Description |
|-----------|------|:--------:|-------------|
| `campaignId` | string | ✓ | Campaign ID (`cmp_xxx`) |

Replaces the legacy `pause_campaign` tool which used the deprecated PATCH-status alias. Emits a full audit event with `lifecycle_op_id` correlation.

## lifecycle.stop_campaign (Stage 2) <StabilityBadge level="stable" />

Stop a campaign — drain or force.

| Parameter | Type | Required | Description |
|-----------|------|:--------:|-------------|
| `campaignId` | string | ✓ | Campaign ID (`cmp_xxx`) |
| `mode` | `"drain" \| "force"` | ✓ | drain (recommended) or force (immediate) |
| `confirm_force` | `true` | * | **REQUIRED when `mode === "force"`** |

- **`drain`** — flips to `stopping`; the BullMQ sweeper promotes to `stopped` once in-flight enrollments finish naturally. Reversible until promoted.
- **`force`** — ⚠️ DESTRUCTIVE. Cancels all pending wait jobs immediately and force-exits in-flight enrollments. In-flight emails already sent to Resend will still go out (BullMQ semantics — Stage 1 design).

> "Stop the welcome campaign gracefully — let the in-flight enrollments finish."
```
Tool: lifecycle.stop_campaign
{ "campaignId": "cmp_xxx", "mode": "drain" }
```

> "Cancel the welcome campaign immediately."
```
Tool: lifecycle.stop_campaign
{ "campaignId": "cmp_xxx", "mode": "force", "confirm_force": true }
```

---

## lifecycle.archive_campaign (Stage 2) <StabilityBadge level="stable" />

⚠️ **TERMINAL** — archive a campaign.

| Parameter | Type | Required | Description |
|-----------|------|:--------:|-------------|
| `campaignId` | string | ✓ | Campaign ID (`cmp_xxx`) |
| `confirm_terminal` | `true` | ✓ | **REQUIRED**. Acknowledge that archive is irreversible. |

Archived campaigns CANNOT be reactivated. Used to retire campaigns from the active list while preserving the audit trail and historical analytics. Idempotent on already-archived campaigns.

> "Retire the welcome campaign — it's been replaced by the v2 onboarding flow."
```
Tool: lifecycle.archive_campaign
{ "campaignId": "cmp_xxx", "confirm_terminal": true }
```
