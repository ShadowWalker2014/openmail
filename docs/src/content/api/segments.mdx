# Segments API

Segments are dynamic groups of contacts defined by rules. When a contact's attributes change, they automatically enter or exit matching segments.

**Base URL:** `/api/v1/segments`

## List Segments

<EndpointBadge method="GET" path="/api/v1/segments" />

Returns all segments in the workspace.

```bash
curl -H "Authorization: Bearer om_..." \
  https://your-api/api/v1/segments
```

## Create Segment

<EndpointBadge method="POST" path="/api/v1/segments" />

Creates a new segment with filter conditions.

### Request Body

| Field | Type | Required | Description |
|-------|------|:--------:|-------------|
| `name` | string | ✓ | Segment name |
| `conditions` | array | ✓ | List of filter conditions |
| `conditionLogic` | `"and"` or `"or"` | — | How to combine conditions (default: `"and"`) |
| `description` | string | — | Optional description |

### Condition Object

| Field | Type | Description |
|-------|------|-------------|
| `field` | string | Field to evaluate, e.g. `"attributes.plan"`, `"email"` |
| `operator` | string | Comparison operator (see below) |
| `value` | string, number, or boolean | Value to compare against (not required for `exists` / `not_exists`) |

### Operators

| Operator | Description |
|----------|-------------|
| `eq` | Equals |
| `ne` | Not equals |
| `gt` | Greater than |
| `gte` | Greater than or equal |
| `lt` | Less than |
| `lte` | Less than or equal |
| `contains` | String contains |
| `not_contains` | String does not contain |
| `exists` | Field exists (has any value) |
| `not_exists` | Field does not exist |

### Examples

**All Pro plan users:**

```bash
curl -X POST https://your-api/api/v1/segments \
  -H "Authorization: Bearer om_..." \
  -H "Content-Type: application/json" \
  -d '{
    "name": "Pro Users",
    "conditions": [
      { "field": "attributes.plan", "operator": "eq", "value": "pro" }
    ]
  }'
```

**Enterprise users with MRR > $500:**

```bash
curl -X POST https://your-api/api/v1/segments \
  -H "Authorization: Bearer om_..." \
  -H "Content-Type: application/json" \
  -d '{
    "name": "High-Value Enterprise",
    "conditionLogic": "and",
    "conditions": [
      { "field": "attributes.plan", "operator": "eq", "value": "enterprise" },
      { "field": "attributes.mrr", "operator": "gt", "value": 500 }
    ]
  }'
```

**Any contact with a company attribute (any value):**

```bash
{
  "name": "Contacts with Company",
  "conditions": [
    { "field": "attributes.company", "operator": "exists" }
  ]
}
```

## Delete Segment

<EndpointBadge method="DELETE" path="/api/v1/segments/:id" />

Permanently deletes a segment. Broadcasts and campaigns using this segment will lose their reference.

## Segment Object

| Field | Type | Description |
|-------|------|-------------|
| `id` | string | Unique ID (`seg_` prefix) |
| `name` | string | Segment name |
| `description` | string or null | Optional description |
| `conditions` | array | Filter conditions |
| `conditionLogic` | string | `"and"` or `"or"` |
| `createdAt` | timestamp | Creation time |
| `updatedAt` | timestamp | Last update time |
