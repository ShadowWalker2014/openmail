# Broadcasts API

Broadcasts are one-off email blasts sent to one or more contact segments. A broadcast starts as a draft, then can be sent immediately or scheduled for a future time.

**Base URL:** `/api/v1/broadcasts`

## List Broadcasts

<EndpointBadge method="GET" path="/api/v1/broadcasts" />

Returns all broadcasts in the workspace.

```bash
curl -H "Authorization: Bearer om_..." \
  https://your-api/api/v1/broadcasts
```

Creates a new broadcast in `draft` status. Either `htmlContent` or `templateId` must be provided.

## Update Broadcast

<EndpointBadge method="PATCH" path="/api/v1/broadcasts/:id" />

Updates a broadcast that is still in `draft` status. Cannot update a sending or sent broadcast. All fields are optional â€” only provided fields are updated.

Pass `scheduledAt` to schedule (or reschedule) the broadcast. Pass `scheduledAt: null` to clear a previously set schedule and return to draft.

### Request Body

All fields from Create Broadcast are accepted. Additionally:

| Field | Type | Description |
|-------|------|-------------|
| `scheduledAt` | ISO datetime or `null` | Set or clear the scheduled send time |

```bash
# Schedule a broadcast
curl -X PATCH https://your-api/api/v1/broadcasts/brd_abc123def456 \
  -H "Authorization: Bearer om_..." \
  -H "Content-Type: application/json" \
  -d '{ "scheduledAt": "2025-03-15T09:00:00Z" }'

# Clear the schedule (back to draft)
curl -X PATCH https://your-api/api/v1/broadcasts/brd_abc123def456 \
  -H "Authorization: Bearer om_..." \
  -H "Content-Type: application/json" \
  -d '{ "scheduledAt": null }'
```

## Send Broadcast

<EndpointBadge method="POST" path="/api/v1/broadcasts/:id/send" />

Immediately queues the broadcast for sending. This action is irreversible.

```bash
curl -X POST https://your-api/api/v1/broadcasts/brd_abc123def456/send \
  -H "Authorization: Bearer om_..."
```

Status immediately transitions to `sending`, then to `sent` once all emails are queued. Send progress is tracked via `sentCount` / `recipientCount`.

## Test Send

<EndpointBadge method="POST" path="/api/v1/broadcasts/:id/test-send" />

Sends a single test email to a specified address. Does not affect broadcast status or send counts. Useful for previewing the email before sending to the full segment.

```bash
curl -X POST https://your-api/api/v1/broadcasts/brd_abc123def456/test-send \
  -H "Authorization: Bearer om_..." \
  -H "Content-Type: application/json" \
  -d '{ "email": "you@example.com" }'
```

## List Sends

<EndpointBadge method="GET" path="/api/v1/broadcasts/:id/sends" />

Returns paginated email send records for the broadcast. Each record represents one email sent to one contact.

### Query Parameters

| Parameter | Type | Default | Description |
|-----------|------|---------|-------------|
| `page` | integer | `1` | Page number |
| `pageSize` | integer | `50` | Results per page (max 500) |
| `status` | string | â€” | Filter by status: `queued`, `sent`, `failed`, `bounced` |

```bash
curl -H "Authorization: Bearer om_..." \
  "https://your-api/api/v1/broadcasts/brd_abc123def456/sends?status=failed"
```

## Top Links

<EndpointBadge method="GET" path="/api/v1/broadcasts/:id/top-links" />

Returns the most clicked links in a sent broadcast, ranked by click count.

```bash
curl -H "Authorization: Bearer om_..." \
  https://your-api/api/v1/broadcasts/brd_abc123def456/top-links
```

Response:
```json
[
  { "url": "https://example.com/features", "clicks": 142 },
  { "url": "https://example.com/pricing", "clicks": 87 }
]
```

## Delete Broadcast

<EndpointBadge method="DELETE" path="/api/v1/broadcasts/:id" />

Permanently deletes a draft or scheduled broadcast. Cannot delete a broadcast in `sending` or `sent` status.

Returns `200 { "success": true }` on success.

## Broadcast Statuses

| Status | Description |
|--------|-------------|
| `draft` | Created but not yet sent or scheduled |
| `scheduled` | Set with a `scheduledAt` in the future |
| `queued` | Submitted to the send queue |
| `sending` | Currently being processed by workers |
| `sent` | All emails delivered to the queue |
| `failed` | Send failed â€” check worker logs |

## Broadcast Object

| Field | Type | Description |
|-------|------|-------------|
| `id` | string | Unique ID (`brd_` prefix) |
| `name` | string | Internal broadcast name |
| `subject` | string | Email subject line |
| `status` | string | Current status (see above) |
| `segmentIds` | string[] | Target segment IDs |
| `htmlContent` | string or null | Inline HTML content |
| `templateId` | string or null | Referenced template ID |
| `fromEmail` | string or null | Sender address |
| `fromName` | string or null | Sender name |
| `recipientCount` | integer | Total unique recipients |
| `sentCount` | integer | Emails sent so far |
| `scheduledAt` | timestamp or null | Scheduled send time |
| `sentAt` | timestamp or null | When the broadcast finished sending |
