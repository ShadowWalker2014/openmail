# Analytics API

The Analytics API provides aggregate performance metrics for your workspace and individual broadcasts.

**Base URL:** `/api/v1/analytics`

<Callout type="note">
Open and click rates are returned as **percentages** (e.g. `24.3` means 24.3%), not fractions. This is different from some other analytics APIs.
</Callout>

## Workspace Overview

<EndpointBadge method="GET" path="/api/v1/analytics/overview" />

Returns workspace-wide metrics for the last 30 days.

```bash
curl -H "Authorization: Bearer om_..." \
  https://your-api/api/v1/analytics/overview
```

### Response

```json
{
  "contacts": 12453,
  "sends": 58920,
  "opens": 14317,
  "clicks": 3002,
  "unsubscribes": 118,
  "openRate": 24.3,
  "clickRate": 5.1,
  "period": "30d"
}
```

### Response Fields

| Field | Type | Description |
|-------|------|-------------|
| `contacts` | integer | Total contacts in workspace |
| `sends` | integer | Emails sent in the last 30 days |
| `opens` | integer | Unique opens tracked in the last 30 days |
| `clicks` | integer | Unique clicks tracked in the last 30 days |
| `unsubscribes` | integer | Unsubscribes in the last 30 days |
| `openRate` | float | Open percentage, e.g. `24.3` = 24.3% |
| `clickRate` | float | Click percentage, e.g. `5.1` = 5.1% |
| `period` | string | Period covered: `"30d"` |

## Broadcast Analytics

<EndpointBadge method="GET" path="/api/v1/analytics/broadcasts/:id" />

Returns performance metrics for a specific broadcast.

```bash
curl -H "Authorization: Bearer om_..." \
  https://your-api/api/v1/analytics/broadcasts/brd_abc123def456
```

### Response

```json
{
  "broadcastId": "brd_abc123def456",
  "sentCount": 4812,
  "openCount": 1169,
  "clickCount": 245,
  "openRate": 24.3,
  "clickRate": 5.1,
  "open": 1169,
  "click": 245,
  "unsubscribe": 12,
  "bounce": 9
}
```

### Response Fields

| Field | Type | Description |
|-------|------|-------------|
| `broadcastId` | string | Broadcast ID |
| `sentCount` | integer | Emails sent |
| `openCount` | integer | Total open events (from broadcast record) |
| `clickCount` | integer | Total click events (from broadcast record) |
| `openRate` | float | Open percentage, e.g. `24.3` = 24.3% |
| `clickRate` | float | Click percentage, e.g. `5.1` = 5.1% |
| `open`, `click`, `bounce`, etc. | integer | Live event counts by type from the tracker |

The response also includes dynamic keys — one per email event type (`open`, `click`, `bounce`, `complaint`, `unsubscribe`) — aggregated from the tracker in real time.

## Tracking Notes

Open and click tracking require the `tracker/` service to be deployed and `TRACKER_URL` set in the API environment. Without the tracker:
- `openCount` and `clickCount` will be `0`
- `openRate` and `clickRate` will be `0`
- All `sentCount` and send-level fields are always accurate

Open tracking works by embedding a 1×1 transparent pixel in each email. Some email clients block pixels, so actual open rates may be higher than reported.
