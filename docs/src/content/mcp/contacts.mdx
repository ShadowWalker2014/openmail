# Contact Tools

5 tools for managing your contact database and tracking customer activity.

## list_contacts

List contacts with optional search and pagination.

**Parameters:**

| Parameter | Type | Required | Description |
|-----------|------|:--------:|-------------|
| `page` | number | ‚Äî | Page number (default: 1) |
| `pageSize` | number | ‚Äî | Results per page (default: 50) |
| `search` | string | ‚Äî | Filter by email (partial match) |

**Example:**

> "List all contacts whose email contains `@acme.com`"

```
Tool: list_contacts
{ "search": "@acme.com" }
```

## create_contact

Create or update a contact by email (upsert). If a contact with the email already exists, it is updated.

**Parameters:**

| Parameter | Type | Required | Description |
|-----------|------|:--------:|-------------|
| `email` | string | ‚úì | Contact email address |
| `firstName` | string | ‚Äî | First name |
| `lastName` | string | ‚Äî | Last name |
| `attributes` | object | ‚Äî | Custom key-value traits |

**Example:**

> "Create a contact for bob@example.com with plan: pro and company: Acme"

```
Tool: create_contact
{
  "email": "bob@example.com",
  "firstName": "Bob",
  "attributes": { "plan": "pro", "company": "Acme" }
}
```

## track_event

Track a customer event. This creates or upserts the contact and fires any active campaigns listening for this event name.

**Parameters:**

| Parameter | Type | Required | Description |
|-----------|------|:--------:|-------------|
| `email` | string | ‚úì | Contact email address |
| `name` | string | ‚úì | Event name in snake_case |
| `properties` | object | ‚Äî | Event-specific data |

**Example:**

> "Track a plan_upgraded event for alice@example.com with to_plan: enterprise and mrr: 499"

```
Tool: track_event
{
  "email": "alice@example.com",
  "name": "plan_upgraded",
  "properties": {
    "from_plan": "pro",
    "to_plan": "enterprise",
    "mrr": 499
  }
}
```

This call will:
1. Create the contact if it doesn't exist
2. Record the event
3. Fire any active campaigns with `triggerType: "event"` and `eventName: "plan_upgraded"`
